プログラマーは普通、
UNIX SystemVコマンド・ノートやシステムについてくるマニュアルに準じて、使用方法を書いてます。
（後で、誰が見ても分かるように　＆　後で自分が見ても分かるように(^^;;　）
たとえば、トップには

コマンド名：なにするコマンドか
USAGE：
Command  [-sar][name]

-s：総計ブロックの報告
-a：各ファイルに対する情報出力
-r：読みとれないディレクトリ、...
name：ファイルあるいはディレクトリ...

あと、印刷したとき分からなくならないようにファイル名を書いたり、
サーバを変えても困らないように動作したＯＳ名とか書きます。
作成日や作者も書きますね。


途中には、次の行からは何をやろうとしているのかを書きます。
（一行ごとってわけではなく機能毎に書きます）
これも大変、重要ですね。
これを書いておいてから、間にコーディングしたりします。

では、いいものを作って公開して下さい。（^^）
