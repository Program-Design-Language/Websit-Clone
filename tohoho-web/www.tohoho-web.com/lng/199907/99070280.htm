FrontPage 2000でホームページ作成しています。
プロバイダはRimnetですが、CGIを動かすためには.htaccessをサーバー上に置く必要があります。
ところが、FrontPage 2000を使ってファイルをアップロードすると、サーバー上の.htaccessを消して良いかと毎回聞いてきます。PCの中には.htaccessは置いてあるのですが、なぜか認識しないようです。試しに、.htaccessを他のフォルダからインポートしようとしたら、インポート不可能でした。
FrontPate 98ではこのようなことはなかったのですが、どなたか解決方法をご存知ではないでしょうか。

ちなみに、FrontPate 98までは、PC側のファイルに追加・変更があればそのファイルをアップロードしていましたが、PC側のファイルを削除してもサーバー側のファイルは影響を受けませんでした。ところが、2000になって、PC側のファイルを削除するとサーバー側のファイルも削除するようになっています（もちろん確認はしてきますが）。すべての操作をFrontPageで行えば便利な機能でしょうが、ちょっとおせっかいです。例えば、手動でアップロードしたCGIファイルなど、消されてしまう可能性があります。
