きっと、サーバの仕様なのでしょう。
CGI がオーナー権限で動くなら 700 でいい（pos.to ネットなら？）
一般的には、CGI は nobody から使われるので 707 じゃないと無理。

B-Cus 1999/07/12(月) 20:09:25
700で動くならそれが一番。

CGIがnobody権限で動いていて、cgi-bin/の下のファイルに
アクセスするなら701。
CGIがnobody権限で動いていて、cgi-bin/の下のファイル一覧を
取得してるなら705。
CGIがnobody権限で動いていて、cgi-bin/の下にファイルを
新規作成するなら707。

777にする必要はまずないと思います。
